# 2网络编程步骤
1. 调用Socket函数创建套接字
2. 调用bind函数分配IP地址和端口号
3. 调用listen函数转为可接受请求的状态
4. 调用accept函数受理连接请求

网络传输速率（带宽）---吞吐量
实际上这二者是相似的概念，单位也都是bps。这个传输速率指的不是在链路上的移动速度而是一秒中可以传输多少个比特，相当于是并行传输多少个比特，而吞吐量则是更为广泛的概念，也可以代表了计算机CPU处理这些数据的速率。

- 硬件设备
1. 网卡（NIC，网络接口卡） 这是链接网络的关键设备，必须有网络接口卡才可以连接同轴电缆或是通过无线网连接网络。
2. 中继器：就是用来放大衰减信号的设备，可以延长一下物理链路的长度，并且不是可以无限延长的，网络传输速率越快，可以连接的中继器数量越少。
3. 集线器：可以当成一个多端口的中继器。
4. 网桥：链路层设备，产生的第一个区别就是它可以存储这些数据帧了，然后可以对这个数据帧进行校验，对于出错的就丢弃。其目标就是把收到的数据帧转发到正确的端口，注意端口只是个出口，没有mac地址，是整个交换机才有一个mac地址，这是因为mac和NIC是一一对应的，代表一个可通信的终端，而交换机本身才是一个可通信的终端，出口只是连着的一个线，因此没有mac地址。自学式网桥是通过记录收到的数据帧来记录设备所在的转发端口的，比如a发了个包发到1口，那么下次目的地址为a的时候就从1发出去。
5. 交换集线器相当于一个多端口网桥。这个就是上面说的那种自学式网桥的所属类别，实际上传统网桥相当于只有两个口，比起中继器估计就多个存储校验的功能。
6. 路由器：不同于网桥通过mac地址进行转发，路由器通过ip地址进行转发。
7. 网关：这里要区别广义的网关和默认网关之间的区别：
    广义的网关是用于跨越不同协议，不同层级，不同格式的转换设备。
    而狭义的默认网关一般指路由器，就是你你的内网到外网的出口，可以理解为第三层的一个网关设备，可以转换你内网的协议和外网的协议（个人理解）
    常见的其他网关设备：
    代理服务器 HTTP转发，缓存 应用层
    NAT网关 私网IP到公网IP地址的转换 网络层传输层混合设备
    API网关 管理API，权限，限流 应用层
    邮件网关 不同邮件协议之间的转换 应用层
